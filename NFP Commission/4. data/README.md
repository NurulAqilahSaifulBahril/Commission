# NFP Commission Calculator

Python report for **Net Floor Price (NFP)** agent commission on paid invoices.

## Setup

```bash
pip install -r requirements.txt
set POSTGRES_PROXY_TOKEN=your_bearer_token_here
python nfp_commission.py --year 2026
```

Output (all generated by `nfp_commission.py`):
- **Console** ‚Äî grid tables printed when you run the script
- **`nfp_commission_report.txt`** ‚Äî same tables as text
- **`nfp_commission_report.csv`** ‚Äî same columns as the table (opens in Excel)
- **`nfp_commission_report.html`** ‚Äî same tables in a browser

Optional: `--full-csv path.csv` for all internal fields (audit).

## Filters (defaults)

| Rule | Implementation |
|------|----------------|
| Paid | `invoice.paid = TRUE` |
| Full payment in 2026 | `EXTRACT(YEAR FROM full_payment_date) = 2026` |
| Internal full-time agent | `agent.agent_type` in (`internal`, `FULL TIME`) |

Duplicate `invoice_number` rows are collapsed to **one row**; **EPP** is taken once per invoice number (max non-zero line EPP or `effective_epp` across duplicates, not summed).

**Phase (single vs three)** comes from `public.seda_registration.phase_type`, linked via `invoice.linked_seda_registration` or `seda_registration.linked_invoice`.

## Formulas

| Field | Formula |
|-------|---------|
| **Sales Price** | `total_amount - epp_cost` (RM) |
| **System Price** | Package line `unit_price` (`invoice_item` where `is_a_package = true`) |
| **EPP interest** | Sum of `invoice_item.epp`, else EPP Interest line items (`invoice_item.description`), else `invoice.effective_epp` |
| **Commission a** | `(Sales Price - NFP) √ó 25%` when Sales Price > NFP |
| **Commission b** | `(System Price - NFP) √ó 100%` when System Price > NFP (audit field only) |
| **Commission c** | `(NFP - Sales Price) √ó 20%` when Sales Price < NFP (deducted) |
| **NFP Commission** | `a - c` (sales price vs NFP only) |

## Net Floor Price lookup

| Invoice date | Panel rating | Source |
|--------------|--------------|--------|
| Before 1 Oct 2025 | any | No NFP (commission = 0) |
| Oct‚ÄìDec 2025 | 620W | `data/nfp_620w_schedule.json` (from PDF) |
| Jan 2026 | 650W | Uses **FEB 2026** sheet |
| Feb‚ÄìMay 2026 | 650W | `1. Excel/STRING 650W package STRING INVERTER.xlsx` (by invoice month). **Uses this even if the package line says 620W.** |
| TNG rebate on invoice | ‚Äî | Detected from line items (`TNG`, `TNG Cash Option`, `Swap TNG`, Road Show + TNG, etc.). Uses **FINAL PRICE AFTER DISCOUNT (WITH TNG REBATE)** when Yes. See `tng_audit_2026.csv`. |
| **NFP rounding (Excel rebuild)** | ‚Äî | **FEB** sheet (e.g. FEB 2026, used for Jan‚ÄìFeb invoices): half-up to **2 decimal places** (sen). **Other** monthly sheets: **1 decimal** rule (e.g. 20,135.75 ‚Üí 20,135.6). Rebuilt from rounded ‚ÄúPrice after Max % Discount‚Äù minus rebates. |

Panel count: `invoice.panel_qty`, or parsed from package description (e.g. `20X Jinko‚Ä¶`).  
If rating is missing for post‚ÄìOct 2025 invoices, **650W** is assumed.
